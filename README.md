# ClientFlow

## Обзор
ClientFlow — это современная CRM-платформа для управления лидами, ориентированная на малый бизнес и отделы продаж. Решение помогает оцифровать воронку продаж, организовать работу с потенциальными клиентами и использовать возможности ИИ для продуктивного анализа коммуникаций.

На сегодняшний день проект представляет собой функциональный прототип, ориентированный на демонстрацию возможностей продукта в стабильной среде. Архитектура спроектирована как **demo-first**: основной упор сделан на качественную презентацию функционала без барьеров в виде регистрации. Оптимальный способ ознакомления с платформой — это выделенный демонстрационный путь.

## Live Demo / Как лучше смотреть проект
Основной маршрут для оценки проекта — демонстрационный режим. Это позволяет мгновенно увидеть продукт в "наполненном" состоянии с реалистичными данными и активной аналитикой.

1. Откройте главную страницу сайта.
2. Нажмите кнопку **View Demo**.
3. Система перенаправит вас в раздел `/demo/dashboard`.

Этот путь рекомендуется, так как он обходит этап авторизации и сразу демонстрирует CRM-таблицы, KPI и ИИ-функции на подготовленных кейсах. Это экономит время проверяющего и исключает проблему "пустой базы данных".

## Что реализовано сейчас
- **Управление лидами (CRM):** Полноценная работа с карточками клиентов, изменение статусов и управление воронкой.
- **Заметки и логи активности:** История взаимодействий внутри профиля каждого лида с поддержкой хронологического списка.
- **Аналитическая панель:** Визуализация метрик воронки, конверсии и стоимости пайплайна с использованием динамических графиков.
- **ИИ-инструменты:** Интеграция с Google Gemini для автоматического резюмирования истории лида и генерации контекстных ответов.
- **Экспорт данных:** Функциональная выгрузка списков лидов в формате CSV для внешней обработки.
- **Биллинг и Онбординг:** Имитация процесса выбора тарифного плана и первичной настройки аккаунта.
- **Режим Showcase:** Специальная обертка приложения для работы в режиме "демо" параллельно с основной версией.
- **Честные границы:** Действия внутри `/demo` являются симуляциями (не изменяют БД), тогда как основной `/dashboard` требует авторизации.

## Технологии
- **Frontend/Fullstack:** Next.js 15 (App Router), React 19, TypeScript.
- **Стилизация:** Tailwind CSS v4, Motion (анимации), Lucide React (иконки).
- **Управление данными:** SWR (валидация данных), Recharts (визуализация).
- **ИИ-слой:** SDK Google Generative AI (@google/genai).
- **Persistence:** Supabase (PostgreSQL) + локальное JSON-хранилище (fallback).

## Архитектура
Проект построен на **Next.js App Router** с акцентом на модульность и расширяемость.
- **API Layer:** Использование Route Handlers для создания типизированного REST-интерфейса.
- **Repository Pattern:** Слой доступа к данным абстрагирован, что позволяет приложению бесшовно переключаться между облачной БД и локальным моком.
- **Demo-first Routing:** Поддержка параллельных веток интерфейса (`/dashboard` vs `/demo`), использующая React Context для внедрения стабильного демонстрационного состояния.

## Showcase / Demo Mode
Раздел `/demo` — это не просто набор скриншотов, а интерактивная копия приложения. Она существует для того, чтобы показать возможности CRM без необходимости ручного ввода данных. В отличие от стандартного пути пользователя, здесь фокус смещен на консистентность презентации: вы видите заполненные графики, сложную историю заявок и работу ИИ на реальных примерах прямо "из коробки".

## Текущие ограничения
- **Демо-приоритет:** Основная ценность проекта сейчас сосредоточена в маршруте `/demo`.
- **Имитация платежей:** Модуль биллинга является визуальной демонстрацией и не проводит реальных транзакций.
- **Защищенные действия:** Некоторые операции в демо-режиме намеренно ограничены (Read-Only/Simulated) для сохранения стабильности презентации.
- **Статус проекта:** Это инженерное портфолио, демонстрирующее навыки проектирования и разработки, а не готовый SaaS-бизнес.

## Локальный запуск
1. **Установка зависимостей:**
   ```bash
   npm install
   ```
2. **Переменные окружения:**
   Создайте файл `.env.local`:
   ```env
   # Обязательно для функций ИИ
   NEXT_PUBLIC_GEMINI_API_KEY=your_key_here
   
   # Опционально для регистрации/авторизации
   JWT_SECRET=your_secret_string
   ```
3. **Запуск сервера разработки:**
   ```bash
   npm run dev
   ```
Для ознакомления с `/demo` достаточно только ключа Gemini. Основной дашборд может потребовать настройки Supabase.

## Рекомендуемый сценарий просмотра
1. Перейдите на лендинг (оцените подачу и визуальный стиль).
2. Нажмите **View Demo** (вход в showcase-среду).
3. Перейдите в раздел **Leads** и откройте любую карточку клиента.
4. Попробуйте функции **AI Summary** или **Generate Reply**.
5. Оцените визуализацию данных в разделе **Analytics**.
